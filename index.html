The Ed-Fi Alliance publishes the Ed-Fi data standard, a defined set of rules that allows educational data systems to connect, and technology that enables real world use of the data standard. This primarily happens through an operational data store (ODS) and accompanying API.

You can read more about Ed-Fi here: https://www.ed-fi.org/

This tutorial will walk you through how to create a Ed-Fi sandbox environment in Google Cloud including the ODS and the API.

Before you can move forward in this tutorial, you will need to request access to our Ed-Fi sandbox image in Google Cloud. Please complete the Google Form below.

https://docs.google.com/forms/d/e/1FAIpQLSdILnko_UZ-iuLNymH7dqnMi7MEiVkRSKQ2Br1RMfyGL-yIjA/viewform

We've given you access to our Ed-Fi sandbox image and you're ready to build the virtual machine in your Google Cloud. Let's get started! Log into your Google Cloud and navigate to Compute Engine, or click below:

https://console.cloud.google.com/compute/instances

Activate Cloud Shell

Look for the Cloud Shell icon shown above in your Google Cloud and click it to activate a Cloud Shell connection. Paste in the command below to create the virtual machine.

gcloud compute instances create sandbox --image edfi-dev-311 --image-project public-resources-240800 --zone us-central1-c

It should take ~5 minutes for Google to create your new Compute Engine VM. You'll be able to see that the process has started by clicking on the button. When you see a green check mark next to your new VM, head to the next step in this tutorial.

The first thing we want to do now that you have your Compute Engine VM is to reset the password for Windows. Click on the arrow next to RDP and select Set Windows password.

When prompted, enter edfi as your username and click Set. Google will generate a new password for you, please store it somewhere safe.

Chrome RDP

Install the Chrome RDP for Google Cloud Platform extension.

Once installed, head back to your VM instances page in Google Cloud and click on RDP. A window should open asking you for your recently set username and password.

Click on the Start Menu and launch Microsoft SQL Server Management under the Microsoft SQL Server Tools 17 folder. You'll see a few databases setup and ready to go. The database EdFi_Ods_Sandbox_GapYwJToOmLFuxj20cgyf is seeded with fake school data so you can get a good idea of what an Ed-Fi database looks like.

Head to the Start Menu and launch Visual Studio 2015. Click on File, Recent Projects and Solutions, and launch the recently opened solutions file. Once loaded, click Start on the toolbar.

It may take a few minutes for Visual Studio to launch the project. Once launched, open Google Chrome and head to http://localhost:38928/. After a few minutes, you'll see the Sandbox Administration portal. Click Login and login with:

https://techdocs.ed-fi.org/display/ODSAPI31/Using+the+Sandbox+Administration+Portal

You will want to change the password to you Sandbox Administration Portal. This can be done by stopping the Visual Studio project and editing the file AdminCredential in C:\Ed-Fi-ODS-Implementation\Application\EdFi.Ods.Admin.Web
